CodeSentinel V2 Change Notes
===========================

Project:
CodeSentinel

Purpose of V2:
Make the app more stable, more helpful in debugging mode, and more polished in frontend behavior.

Main changes added in V2:

1. Debug and Fix mode split improved
- Debug mode now starts a coaching flow instead of giving the answer directly.
- Fix mode now returns corrected code directly.
- Mode buttons now switch active style correctly.

2. Progressive debugging flow improved
- After Run, the app asks where the user thinks the bug is.
- Send Thought now checks whether the user correctly found the bug.
- If the bug is found, the app says: Good Job you found the bug now try to debug it.
- If the corrected code is provided, the app completes the session automatically.

3. Repeated message loop fixed
- After the bug is already identified, later thoughts no longer repeat the same canned line.
- The backend now gives a fresh next-step coaching response.

4. Status system improved
- Initial state is IDLE.
- While waiting for Gemini, status changes to RUNNING.
- After response arrives, status changes to OUTPUT.
- On completion, status changes to DONE.
- On request failure, status changes to ERROR.

5. Command support added
- clear / Clear / CLEAR / clr / Clr / CLR now clear the response box.
- /help now shows a help guide.
- Backend also supports these commands so the app still works if frontend cache is stale.

6. Help output improved
- Help output was redesigned for readability.
- Button names like Run, Send Thought, Next Hint, and Fix are highlighted.
- Commands line is styled separately.
- Light mode colors were improved so help text stays visible.

7. Code generation request blocking added
- Requests like write code, generate code, create code, build code, etc. are blocked.
- The app now responds with a debugger-only message instead of acting like a code generator.

8. Non-code input handling improved
- Plain text like hi how are you no longer enters the debugging flow.
- The backend now checks whether input looks like real code before starting a session.

9. Chat output styling improved
- Special success and bug-found messages are styled with color.
- Help text styling was improved.
- Light mode color visibility for colored messages was improved.

10. Navbar and layering fixes
- Navbar stacking was raised so scrolling content stays under the glass effect.
- Theme icon button shape was fixed to stay perfectly round.

11. Response panel and light mode polish
- Light mode response box colors were improved.
- Input and panel visibility were improved in light theme.

12. Footer and alignment fixes
- Footer alignment was centered.
- Section alignment regressions caused by footer edits were corrected.

13. Frontend stability improvements
- app.js was cleaned up to reduce quote-related breakages.
- Command handling was moved into clearer helper functions.
- Chat rendering was made safer and more predictable.

14. Backend stability improvements
- Backend request handling was cleaned up.
- Gemini JSON parsing was made more robust.
- Session flow for debugging became more reliable.

Files mainly updated in V2:
- backend/server.py
- frontend/app.js
- frontend/style.css
- frontend/index.html

Verification done:
- Python backend syntax check passed.
- Frontend JavaScript syntax check passed.
