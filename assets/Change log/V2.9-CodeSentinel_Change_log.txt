CodeSentinel V2.9 Change Notes
==============================

Project:
CodeSentinel

Purpose of V2.9:
Move the project from an early guided-debug prototype into a more complete trial-launch build with stronger workflow control, cleaner fix-mode output, API-gated access, and better frontend polish.

Main changes added in V2.9:

1. API Registration system added
- A new API Registration section was added above the main debugger area.
- Users must submit their AI API key before CodeSentinel can start.
- The main debugger stays locked until the API key is accepted.
- A Remove API Key button was added so the key can be cleared and the app can be locked again.

2. Fix mode was redesigned
- Fix mode no longer uses the normal response-and-stats layout.
- The left side now switches into a dedicated Fix layout.
- Fixed code is shown in its own card.
- Change log is shown in a separate card below it.
- This makes direct correction easier to read and keeps explanation separate from output code.

3. Fixed code output improved
- Fixed code now uses syntax-highlighted rendering instead of plain text.
- A copy icon button replaced the older copy text button.
- Copy behavior now works more like a modern code assistant UI.
- Fixed code scroll behavior and rounding were improved to preserve card shape.

4. Change Log box behavior improved
- The Change Log box now has its own stable layout and scrolling behavior.
- Long explanations no longer stretch the whole card or break panel alignment.
- Text wrapping was improved so API error or quota messages stay inside the box.
- Scrollbar styling was adjusted to fit the inner outline better.

5. Debug vs Fix mode layout separation improved
- Stats remain visible only in Debug mode.
- Fix mode hides stats and replaces them with direct output cards.
- Main panel heights were tuned so mode switching feels more stable.
- Response and Controls tab spacing was polished to reduce visual jumps.

6. Response styling improved
- User thoughts and AI replies are now visually easier to distinguish.
- Hint replies were given their own highlighted style.
- Wrong-thought replies were colored more clearly.
- Success messages and bug-found messages were given special output styling.
- Help output styling was improved again for readability.

7. Progress and stats system refined
- Progress updates were tied more closely to whether a thought is correct, close, or wrong.
- Wrong turns now reduce the progress bar instead of always increasing it.
- Already-correct code now fills progress to 100 percent and resets the counters.
- Download Stats and Reset Stats logic was improved.
- Stats panel structure was polished with a more balanced card layout.

8. Thought input behavior improved
- Thought input was upgraded from a simple field into a more responsive expanding box.
- Auto-resize behavior was refined so the box grows and shrinks with typing more naturally.
- Max visible lines behavior was improved before scrolling begins.
- After sending a thought, the box clears and resets itself.

9. Input and panel UI polish improved
- Work Mode, Input Code, and Thought Input were organized into cleaner control cards.
- Run and Clear buttons now sit directly under the code box.
- Send Thought, Next Hint, and I Did It now sit directly under the thought box.
- Button coloring was tuned again:
  - Send Thought = sky blue
  - Next Hint = yellow
  - I Did It = soft green

10. Backend guardrails improved further
- API key routes were added for checking, setting, and removing the live key.
- Non-code inputs are blocked more cleanly.
- Code-generation-style prompts are blocked so CodeSentinel stays focused on debugging.
- Already-clean code is detected and handled with a separate chill-style success reply.
- Fix mode parsing became more reliable with fallback handling for messy AI output.

11. Fix mode generation stability improved
- Backend parsing for fixed code and change logs was improved.
- Fallback extraction logic was added for cases where Gemini returns unexpected formatting.
- Change log extraction became more structured.
- Fix output now fails more gracefully when the API has problems.

12. Scrollbar and corner polish improved
- Input Code, Thought Input, Fixed Code, and Change Log scrolling behavior were refined.
- Rounded corners were preserved better while scrolling.
- Inner scrollbar styling was adjusted to avoid breaking card shapes.

13. API-gated launch workflow prepared
- The project now behaves more like a user-facing trial build.
- The software no longer starts fully open by default.
- Users must unlock the app through API registration before using Debug or Fix.
