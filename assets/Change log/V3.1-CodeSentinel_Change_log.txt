CodeSentinel V3.1 Beta Change Notes
===================================

Project:
CodeSentinel

Purpose of V3.1 Beta:
Expand CodeSentinel from a single-provider beta into a gated private beta with multi-provider AI access, better onboarding, and stronger release control for invited testers.

Main changes added in V3.1 Beta:

1. Multi-provider API support added
- CodeSentinel no longer depends on Gemini only.
- Users can now choose from Gemini, OpenAI, Grok, Claude, and DeepSeek.
- The selected provider changes both validation flow and backend request routing.

2. API Registration redesigned
- The API setup area was rebuilt into a cleaner provider-first experience.
- Users now pick a provider visually before entering their API key.
- Provider cards, logo styling, helper text, and selection feedback were all improved for a clearer onboarding flow.

3. Real API access shortcuts added
- A Get Started panel now includes direct links to the real API-key pages for each supported provider.
- This helps testers move from provider choice to key creation faster without leaving the product flow confused.

4. Private beta access gate added
- A Beta Access system now sits in front of API registration.
- Testers must unlock the private beta first before they can submit any provider API key or use the rest of CodeSentinel.
- Access is remembered in the same browser, so invited users do not need to re-enter the beta key every visit.

5. Invite-key system added for controlled rollout
- 25 invite keys were created for limited private testing.
- A master bypass key was added for the owner.
- Invite keys can be terminated later from the registry file, which gives better control over early access and link spreading.

6. Beta tester identity badge added
- The Beta Access panel now shows which tester slot is being used.
- Invite keys display a yellow tester number such as #01 or #12.
- The master key shows a custom developer welcome state instead of a tester number.

7. Keyboard workflow improved
- In Input Code, Shift + Enter now runs the code.
- In Thought Input, Shift + Enter now sends the thought.
- Normal Enter still creates a new line, so writing flow stays natural.
- The same shortcut behavior is now documented in /help.

8. Provider branding and UI polish improved
- Provider logos were updated and cleaned up.
- Active-state accents were tuned by provider brand.
- Gemini received a multi-color border treatment, Claude was tuned to orange, and provider labels/status text were made easier to read.

9. Footer release data updated
- Footer versioning now reflects the V3.1 Beta release.
- The live release timer was reset for this new build so the site shows time elapsed from the V3.1 Beta push.
